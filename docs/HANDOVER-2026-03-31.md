## What's Broken / Needs Fixing

### 1. Hosting CI Job Fails (Edge Runtime)
`@cloudflare/next-on-pages` requires `export const runtime = 'edge'` on all API routes. The tRPC routes and auth routes in `hosting/src/` don't have this. Also `@libsql/client` and `better-auth` may not be Edge-compatible — may need `@libsql/client/web` or Cloudflare D1 instead.

**Fix options:**
- A) Add `runtime = 'edge'` to all route handlers, swap to Edge-compatible DB driver
- B) Use Cloudflare Workers (not Pages) with full Node.js compat flag
- C) Merge `hosting/` into `cloud/` (Hono on Workers, already deployed)

### 2. Two Separate Backend Systems
```
cloud/    → Hono on Cloudflare Workers (api.getsolon.dev) — DEPLOYED
hosting/  → Next.js (app.getsolon.dev) — NOT DEPLOYED
```
Both have auth, billing, instance management. Need to merge into one app.

**Recommendation:** Merge into `cloud/` (Hono Workers) since it's already deployed and Edge-native. Add provisioning routes. Use the `hosting/` React pages as a separate SPA that calls `api.getsolon.dev`.

### 3. DNS for app.getsolon.dev
Not configured yet. Once hosting app deploys:
- If Cloudflare Pages: automatic via `solon-app.pages.dev` CNAME
- If separate SPA: point to Cloudflare Pages project

### 4. Stripe Keys Not Configured
`hosting/src/server/billing/stripe.ts` is ready but no production Stripe keys are set. Need:
